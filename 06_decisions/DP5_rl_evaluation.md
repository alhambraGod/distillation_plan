## 详细分析

### 1. Benchmark 稳定性
- Benchmark 集是否 3 个月内无大改动？
- 多次运行同一版本方差 < 5%？

### 2. 人工评分一致性
- 2 名标注人 Cohen's κ ≥ 0.7？
- 判据文档稳定、业务方没再改过？

### 3. Environment
- agent 运行时接的工具稳定？（没 weekly 更新）
- 能完整复现一次 agent 执行（包括工具返回）？
- 测试环境 vs 生产环境一致？

### 4. Reward
- 候选 reward 函数：
  - [ ] 规则基（如 JSON 有效 + 工具 F1）
  - [ ] LLM judge（如 Claude 打分）
  - [ ] 业务指标（如用户采纳）
  - [ ] 组合
- 是否能自动、快速、稳定给分？
- 是否有"漏洞"（可 hack 的模式）？

### 5. 多步策略问题
- 当前 bad case 是否主要来自"决策路径错"（而不是"格式错"）？
- 多步决策 bad case 占比 ≥ 30%？
- SFT / DPO 是否尝试过但无法解决？

### Option A：不启动 RL
- 大多数条件未满足 → 继续用 SFT/DPO，持续完善 benchmark 和 reward

### Option B：启动 RL 前置调研
- 全部条件满足
- 第一步：调研 GRPO vs PPO，选方案
- 第二步：小规模 POC（50 条样本）
- 第三步：估算成本，决定是否进大规模训练

### Option C：不启动 RL，尝试其他路线
- 某些条件永久无法满足 → 考虑：
  - 更大基座 + SFT
  - Agent workflow 外部优化（拆子任务、加 reflection）
  - 约束解码强化（推理时兜底）
