# 教师模型（Teacher Model）选型深度对比

> 配套阅读：`base_model_selection.md`（学生模型）+ `distillation_techniques.md`（不同蒸馏技术）  
> 本文目标：选"教什么模型来教小模型"。

---

## ⚠️ 重要前提：Claude trace 不是默认可训练数据

需求文档曾默认：用 **Claude 历史 trace** 作为训练数据，这本质是**黑盒蒸馏**（black-box distillation）——只用教师的输出，不用 logits、hidden states。

但 Anthropic 2026-03-16 帮助中心口径明确要求：未经 Anthropic **书面许可**，不得用 Claude 输出训练或开发 AI 模型。课程和项目交付的默认主线因此改为：

1. 已获得书面许可的 Claude trace，可以进入独立数据分区训练。
2. 未获得许可时，使用人工批准 trace、客户自有答案、开源教师输出或开源模型直接微调。
3. Claude 可以作为线上 fallback / judge，但也要受合同和 API 条款约束。

**本文评估的是"是否换成或补充一个开源大模型当教师"**。换教师会带来：

| 维度 | 用 Claude 当教师 | 用开源大模型当教师 |
|---|---|---|
| 数据获取 | ✅ 已有历史 trace | ⚠️ 需要自己跑一遍产数据 |
| 成本 | ✅ 数据沉没成本 | ⚠️ 推理生成 = 花钱 |
| 蒸馏方式 | ❌ 只能黑盒（无 logits） | ✅ 支持白盒（logits/hidden states） |
| 业务对齐 | ✅ 本来就线上跑的就是 Claude | ⚠️ 需要重新验证业务适配 |
| Tokenizer 一致 | ❌ Claude tokenizer 和学生不同 | ✅ 同家族可对齐 |
| License 风险 | ⚠️ Claude ToS 限制 | ✅ 部分模型 Apache 2.0 |
| 天花板 | = Claude 能力 | ≤ 教师能力 |

**推荐策略**（下面会详细展开）：
- **主路线**：用合规数据做黑盒 SFT（已授权 trace / 人工批准答案 / 开源教师输出）
- **加强路线**：用**同家族大模型**（如 Qwen 2.5 72B → Qwen 2.5 7B）做**白盒蒸馏**，增量收益大
- **补充路线**：用开源大模型做**数据增强 / CoT 合成**（补弱项）

## 评分维度

对每个候选模型按以下维度打 1-5 分：

| 维度 | 含义 |
|---|---|
| **综合能力** | 公开 benchmark 平均（MMLU / GSM8K / HumanEval 等） |
| **中文能力** | 中文评测（CMMLU / C-Eval / AlignBench） |
| **工具调用** | function calling 的原生支持和稳定性 |
| **同家族下探** | 是否有对应的小模型学生（同 tokenizer） |
| **白盒友好** | 开源权重 + 架构 + 运行成本 |
| **License** | 商用 / 蒸馏到其他模型的允许度 |
| **推理成本** | 自托管跑教师的 GPU 需求 |
| **稳定性** | 工程生态、bug、社区验证度 |

## 🎯 本项目决策建议

按优先级给三套方案：

### 方案 A：主推 - Qwen 2.5 72B 作教师（白盒蒸馏）

学生：Qwen 2.5 7B Instruct  
教师：Qwen 2.5 72B Instruct  
方法：白盒 KL 散度 + SFT 组合（见 `03_sft/distillation_techniques.md`）

**优势**：
- 同家族无痛对齐
- 白盒蒸馏收益显著（hidden state + logits 信号丰富）
- 72B 自托管可行（4×A100 80G）

**前置条件**：Legal 确认 Qwen 2.5 72B 当前 license 允许本项目的商用、分发和蒸馏用途。

**适用**：算力充裕的团队，想拿到"白盒蒸馏"最佳效果。

### 方案 B：合规黑盒 SFT（默认交付路线）

学生：Qwen 2.5 7B  
教师 / 标签来源：已授权 Claude trace、人工批准答案、开源教师输出三选一  
方法：纯 SFT on 合规 outputs

**优势**：
- 能覆盖已有业务 SOP、工具调用、输出格式
- 不依赖未经许可的闭源 API 输出

**适用**：算力紧张 / 想最快验证 / Legal 尚未放行 Claude 输出训练。

### 方案 C：混合 - 已授权 trace + Qwen 72B 补强

学生：Qwen 2.5 7B  
教师：已授权 Claude trace 或人工批准答案（主数据） + Qwen 2.5 72B（白盒补强 / CoT 合成）  
方法：黑盒 SFT + 白盒 logits 蒸馏 + 合成数据增强

**优势**：
- 兼得两者
- 业务对齐（合规 trace） + 训练信号丰富（白盒）

**适用**：预算和算力都允许；追求最强效果。

**本项目强推 B 作为默认合规路线，A/C 在 license 和书面许可齐备后进入增强阶段**。

## ❗ 重要注意事项

### 1. License 核对必做

所有模型使用前都要 legal 确认：
- 是否允许商用？
- 输出**是否允许用于训练第三方模型**？（这是蒸馏的关键条款）
- 月活 / 营收门槛是否触发？
- 是否需要标注（attribution）？

### 2. 白盒蒸馏的 tokenizer 要求

白盒蒸馏（logit / hidden state 匹配）**要求教师和学生同 tokenizer**：
- Qwen 72B → Qwen 7B：✅
- Llama 70B → Llama 8B：✅
- Qwen 72B → Llama 8B：❌ 不可直接做
- DeepSeek V3 → Qwen 7B：❌ 不可直接做

这是选同家族的关键原因。

### 3. 蒸馏 ≠ 能超越教师

学生的天花板 = 教师的能力水平。所以：
- 教师本身不行 → 再蒸馏也救不了
- 教师擅长 A 不擅长 B → 学生也是
- 教师有 bias → 学生继承 bias

阶段 1 benchmark 一定要**同时测教师的能力**作为上界。
