# Anthropic ToS 合规：Claude 蒸馏的法律风险

> 对应 `project_critique_and_fixes.md` 问题 6。
> ⚠️ **本文不是法律建议**。最终以公司 Legal 团队意见为准。

---

## 1. 核心问题

**我们用 Claude 的历史输出（LangSmith trace）训练小模型，合规吗？**

不同场景答案不同。本文梳理关键条款 + 给出风险档位 + 替代方案。

---

## 2. Anthropic 官方口径（2026-03-16 帮助中心）

Anthropic 官方帮助中心《Can I use my Outputs to train an AI model?》明确写到：

- 客户拥有由自己输入生成的输出，但使用这些输出训练 AI 模型有额外限制。
- 未经 Anthropic **书面许可**，不得使用 Anthropic 服务来训练或开发 AI 模型。
- 明确禁止的例子包括：通用聊天机器人、开放式文本生成模型、把 Claude 输出作为训练目标、逆向工程训练方法。
- 允许的例子主要是不会与 Anthropic 模型竞争的专用工具，例如分类、信息抽取、异常检测、语义搜索等。

课程中的默认假设因此必须改成：

> **除非已有 Anthropic 书面许可或合同补充条款，否则不要把 Claude 输出作为 SFT/DPO/KL 的训练目标。**

参考链接：<https://support.claude.com/en/articles/12326764-can-i-use-my-outputs-to-train-an-ai-model>

## 3. 风险档位分析

### 档位 1：🟢 可做，但要留证

**场景**：
- 用 Claude 输出训练非竞争、窄用途工具，例如分类器、抽取器、异常检测器、语义搜索 reranker
- 或 Anthropic 已给出书面许可 / 合同补充条款，明确允许本项目训练
- 数据已脱敏，且训练用途、部署范围、删除机制都有记录

**判断**：可以进入技术实施，但必须保留合同/许可编号和数据血缘。

### 档位 2：🟡 需 Legal + Anthropic 书面确认

**场景**：
- 用 Claude 输出训练内部 agent 小模型、业务文案模型、开放式文本生成模型
- 模型嵌入对外销售的 SaaS 产品，即使客户不能下载模型
- 模型可能替代 Claude 完成通用对话、写作、分析或 agent 决策

**判断**：不能仅凭“内部使用”认定低风险。必须拿到 Legal 结论，并确认是否需要 Anthropic 书面许可。

### 档位 3：🔴 高风险

**场景**：
- 显式宣传"Claude 替代品"
- 把小模型**包装成通用 LLM**对外卖
- 训完的 adapter / 模型**开源**或客户可下载，并且训练目标来自 Claude 输出
- 未经许可把 Claude 输出作为 SFT/DPO/KL 的训练标签

**判断**：大概率违反 ToS。

## 4. 我们项目的自评

| 问题 | 答案 |
|---|---|
| 有 Anthropic 书面许可允许用输出训练生成式/agent 小模型？ | ❓ 必须确认 |
| 训练目标是否包含 Claude 输出文本或 tool-call trace？ | ✅ 如果用历史 Claude trace |
| 小模型是否开放式生成 / agent 决策模型？ | ✅ |
| 模型是否嵌入产品或对外提供服务？ | ❓ 取决于部署 |
| 对外销售 / 开源模型？ | ❌ 目前假设不做 |
| 宣传为"Claude 替代"？ | ❌ |

**自评档位**：默认 **🟡 需书面确认**。没有 Anthropic 书面许可时，课程主线不得把 Claude 输出作为训练目标。

## 5. Legal Review 流程（强制）

### 5.1 第 1 周必须做的

1. 把本文档提给 Legal
2. 提供以下信息：
   - API 合同版本（企业版 / 团队版 / 个人版）
   - 使用场景说明（内部 / 外部 / 产品化）
   - 小模型部署方式（自托管 / SaaS / 客户端）
   - 数据来源（trace 是否包含 Claude 输出、tool calls、judge 结果）
3. 问 Legal 明确：
   - [ ] 当前 ToS 条款原文
   - [ ] 是否已有 Anthropic 书面许可或合同补充条款
   - [ ] 我们场景是否属于训练/开发 AI 模型
   - [ ] 若属于，是否必须事前和 Anthropic 沟通
   - [ ] 是否需要签补充协议后才能训练
   - [ ] 输出数据保留期限建议

### 5.2 输出文档

Legal 必须出具：
- **白皮书**：评估结论 + 风险档位 + 必要条件
- **签字条款**：技术团队必须遵守的 N 条（如不得开源、不得转售等）

## 6. 替代方案（若未拿到书面许可）

如果不能使用 Claude 输出训练，**本项目不会失败**，因为有替代：

### 替代 A：换教师为开源大模型

**方案**：教师换成 **Qwen 2.5 32B/72B**、**DeepSeek R1 Distill / V3 API** 或 **Llama 3.3 70B** 等，经 license 审核后生成训练数据。

**代价**：
- 需要重新生成训练数据（教师跑一遍已有 prompt）
- 质量可能略低于 Claude（差距 5-15%）
- 自己跑教师推理 GPU 成本 ~$3000

**时间**：多 3-4 周

**参考**：`teacher_model_comparison.md` 方案 A

### 替代 B：直接用已蒸馏的开源模型

**方案**：用 `DeepSeek-R1-Distill-Qwen-7B` 作为起点，在业务数据上继续 LoRA 训练。

**代价**：
- 可能能力不如自己蒸馏
- 但许可证口径更清晰，仍需记录模型卡和 license 版本

**时间**：省 1-2 周（跳过教师推理）

### 替代 C：人工/业务批准 trace

**方案**：把训练目标改为人工审核通过的业务答案、客户已有 SOP 输出、开源教师输出或人工改写后的标准答案。

**代价**：需要业务专家审核，周期比直接 trace 蒸馏长。
**时间**：多 1-3 周，取决于标注吞吐。

### 替代 D：Prompt 工程路线

**方案**：不训练，用线上大模型 + prompt + caching 优化成本。

**代价**：成本降不到目标（40%）
**时间**：最快

## 7. 风险缓解措施（即使合规也要做）

即使拿到许可，建议：

1. **记录数据来源链**：每条训练样本记录教师模型、API 调用时间戳、合同/许可编号
2. **保留撤回能力**：若未来 Anthropic 要求撤回，能快速识别影响的 adapter
3. **定期 re-review**：ToS 可能更新，每季度重新评估
4. **训练数据分源标记**：Claude 许可数据、开源教师数据、人工数据物理隔离
5. **不把未授权 API 输出训产品化生成模型**：所有训练 job 启动前做数据源检查

## 8. 其他教师的合规对比

| 教师 | 合规清晰度 |
|---|---|
| **Claude API** | ⚠️ 默认不能作为生成式/agent 模型训练目标，除非 Anthropic 书面许可 |
| **Qwen 2.5 7B / 14B / 32B** | ✅ 多数 Instruct 模型卡为 Apache 2.0，仍需锁定具体版本 |
| **Qwen 2.5 72B** | ⚠️ Qwen License，商用/分发/蒸馏要 Legal 审 |
| **Qwen 3 dense 版本** | ✅ Apache 2.0 |
| **DeepSeek R1 Distill Qwen** | ✅ MIT，模型卡明确支持商业使用、修改、衍生和蒸馏 |
| **Llama 3.x** | ⚠️ Llama Community License（月活 7 亿限制） |
| **GPT-4 (OpenAI)** | 🔴 ToS 明确禁止训竞品 |
| **豆包 API** | ⚠️ 需 Legal 确认 |
| **开源 Gemma** | ⚠️ Gemma License prohibited use policy |

## 9. 给客户讲的一页纸

**客户**："用 Claude 训小模型合法吗？"

**我们**：
> 最新口径要比以前更谨慎：
> - 没有 Anthropic 书面许可时，不把 Claude 输出作为生成式/agent 小模型训练目标
> - 如果只是非竞争的分类、抽取、异常检测等专用工具，仍要让 Legal 留证
> - 宣传为 Claude 替代 / 开源发布 / 客户下载模型：高风险，不走
>
> 我们课程默认用“已授权 trace / 开源教师 / 人工批准答案”做训练数据。
>
> 如果后续拿到 Anthropic 书面许可，再把 Claude trace 放进独立数据分区并记录来源。
