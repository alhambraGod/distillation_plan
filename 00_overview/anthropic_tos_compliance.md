# Anthropic ToS 合规：Claude 蒸馏的法律风险

> 对应 `project_critique_and_fixes.md` 问题 6。
> ⚠️ **本文不是法律建议**。最终以公司 Legal 团队意见为准。

---

## 1. 核心问题

**我们用 Claude 的历史输出（LangSmith trace）训练小模型，合规吗？**

不同场景答案不同。本文梳理关键条款 + 给出风险档位 + 替代方案。

---

## 2. Anthropic ToS 关键条款

截至 2025-2026 年的 Anthropic Usage Policies 和 Terms of Service 中，和蒸馏相关的关键条款（以官方最新为准，定期 review）：

### 2.1 禁止用途条款

Anthropic 通常禁止：
- 使用输出**创建或训练**竞品（competing）AI 模型
- 逆向工程模型
- 违反可接受使用政策

**关键模糊点**：**"竞品"**的定义。

### 2.2 数据使用条款

- 输入 / 输出的权利归属
- Anthropic 是否可用于自己训练（企业版通常否）
- 客户是否可用于自己目的

### 2.3 企业版 vs 个人版

**企业版（Claude Enterprise / API 付费版）** 通常条款更宽松：
- 客户对输出有更大控制权
- 通常禁止训练**用于销售的**竞品

**个人版 / 免费版** 通常更严格。

---

## 3. 风险档位分析

### 档位 1：🟢 低风险

**场景**：
- 用 Claude 输出训练**内部使用**的小模型
- 不对外销售、不开源模型
- 目的是**成本优化**，不是创造竞品
- 使用的是**企业 API 付费版**

**判断**：多数企业版 ToS 允许。

**本项目初判**：如果我们是付费 API 用户、小模型仅内部使用 → 低风险。

### 档位 2：🟡 中风险

**场景**：
- 用 Claude 输出训练小模型 + 模型**开源发布**
- 或模型**嵌入对外销售的产品**（即使是通过 SaaS 订阅）
- 或被客户下载自部署

**判断**：需要 Legal case-by-case 判断。

### 档位 3：🔴 高风险

**场景**：
- 显式宣传"Claude 替代品"
- 把小模型**包装成通用 LLM**对外卖
- 训完的 adapter / 模型**开源**并注明训练数据来源为 Claude

**判断**：大概率违反 ToS。

## 4. 我们项目的自评

| 问题 | 答案 |
|---|---|
| 使用的是企业 API 付费版？ | ✅ （假设是） |
| 小模型仅内部使用？ | ✅ |
| 对外销售 / 开源模型？ | ❌ |
| 宣传为"Claude 替代"？ | ❌（内部优化用） |
| 是 Anthropic 直接竞品业务？ | ❌（营销 AI / 深度研究工具） |

**自评档位**：🟢 **低风险**，但**必须 Legal 正式背书**。

---

## 5. Legal Review 流程（强制）

### 5.1 第 1 周必须做的

1. 把本文档提给 Legal
2. 提供以下信息：
   - API 合同版本（企业版 / 团队版 / 个人版）
   - 使用场景说明（内部 / 外部 / 产品化）
   - 小模型部署方式（自托管 / SaaS / 客户端）
   - 数据来源（trace 数据内容是否含 Anthropic 所有权内容）
3. 问 Legal 明确：
   - [ ] 当前 ToS 条款原文
   - [ ] 我们场景是否触发任何禁止条款
   - [ ] 是否需要事前和 Anthropic 沟通
   - [ ] 是否需要签补充协议
   - [ ] 输出数据保留期限建议

### 5.2 输出文档

Legal 必须出具：
- **白皮书**：评估结论 + 风险档位 + 必要条件
- **签字条款**：技术团队必须遵守的 N 条（如不得开源、不得转售等）

## 6. 替代方案（若 Legal 判定高风险）

如果 Legal 认为 Claude 蒸馏有风险，**本项目不会失败**，因为有替代：

### 替代 A：换教师为开源大模型

**方案**：教师换成 **Qwen 2.5 72B** 或 **Llama 3.3 70B**（Apache 2.0 / Llama Community License）。

**代价**：
- 需要重新生成训练数据（教师跑一遍已有 prompt）
- 质量可能略低于 Claude（差距 5-15%）
- 自己跑教师推理 GPU 成本 ~$3000

**时间**：多 3-4 周

**参考**：`teacher_model_comparison.md` 方案 A

### 替代 B：直接用已蒸馏的开源模型

**方案**：用 `DeepSeek-R1-Distill-Qwen-7B` 作为起点，在业务数据上继续 LoRA 训练。

**代价**：
- 可能能力不如自己蒸馏
- 但**零合规风险**

**时间**：省 1-2 周（跳过教师推理）

### 替代 C：Prompt 工程路线

**方案**：不训练，用 Claude 本身 + 复杂 prompt + caching 优化成本。

**代价**：成本降不到目标（40%）
**时间**：最快

---

## 7. 风险缓解措施（即使合规也要做）

即使 Legal 通过，建议：

1. **记录数据来源链**：每条训练样本记录 Claude 版本、API 调用时间戳
2. **保留撤回能力**：若未来 Anthropic 要求撤回，能快速识别影响的 adapter
3. **定期 re-review**：ToS 可能更新，每季度重新评估
4. **避免公开声明训练来源**：对外只说"LangSmith 历史数据"，不强调"Claude 输出"
5. **不直接用 API 输出训产品化模型**：训练小模型只用作 fallback / 辅助，不把 Claude 输出 prompt-level 复制

## 8. 其他教师的合规对比

| 教师 | 合规清晰度 |
|---|---|
| **Claude API** | ⚠️ 需 Legal 确认 |
| **Qwen 2.5 ≤ 7B** | ✅ Apache 2.0 无忧 |
| **Qwen 2.5 72B** | ⚠️ 自有协议，商用/蒸馏要读条款 |
| **Qwen 3 dense 版本** | ✅ Apache 2.0 |
| **DeepSeek V3 / R1** | ✅ 宽松商用 + 允许蒸馏 |
| **Llama 3.x** | ⚠️ Llama Community License（月活 7 亿限制） |
| **GPT-4 (OpenAI)** | 🔴 ToS 明确禁止训竞品 |
| **豆包 API** | ⚠️ 需 Legal 确认 |
| **开源 Gemma** | ⚠️ Gemma License prohibited use policy |

## 9. 给客户讲的一页纸

**客户**："用 Claude 训小模型合法吗？"

**我们**：
> 三档风险：
> - 内部使用 + 企业 API + 不对外卖：**低风险**，但需要 Legal 书面确认
> - 模型嵌入销售产品：**中风险**，需要 case-by-case 审核
> - 宣传为 Claude 替代 / 开源发布：**高风险**，大概率违反 ToS
>
> 我们项目属于**低风险**场景，但**第 1 周就会启动 Legal review**。
>
> 如果 Legal 判定风险较高，有两个已准备好的替代方案（换开源教师 / 用 DeepSeek R1 蒸馏版），**不会耽误项目**。
