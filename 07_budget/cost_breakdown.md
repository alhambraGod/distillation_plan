# 成本拆解（Claude vs 小模型）

## 前提

- Claude 3.5 Sonnet：`$3/1M input tokens` + `$15/1M output tokens`
- 假设每次调用平均 2000 prompt + 500 completion tokens
- 每次调用 Claude 成本：`2000 × 3 / 1M + 500 × 15 / 1M = $0.0135`

## 现状（100% Claude）

| 日调用 | 月成本 Claude |
|---|---|
| 1 万 | $4,050 |
| 5 万 | $20,250 |
| 10 万 | $40,500 |
| 50 万 | $202,500 |

## 目标态（80% 小模型 + 20% Claude 兜底）

### 小模型自托管单次成本

vLLM 上单卡 A100 40G，假设 QPS 15、月均利用率 70%：
- 月吞吐：15 × 3600 × 24 × 30 × 0.7 = ~27M 次
- 月硬件成本：$1500
- **单次成本 ≈ $0.00006**（Claude 的 1/225）

**注意**：这是满负载理想值。实际业务 QPS 通常只有几个到十几个，单次成本会上升到 $0.0005-$0.002（仍比 Claude 便宜 10-25 倍）。

### 混合成本（示例：日 10 万调用）

| 层 | 量 | 成本 |
|---|---|---|
| 小模型（80%） | 8 万/日 | 单卡常驻 $1500/月 |
| Claude fallback（20%） | 2 万/日 | 2 万 × 30 × $0.0135 = $8100/月 |
| **合计** | | **$9600/月** |

**相比全 Claude $40500/月，节省 76%**。

### 回本时间（按日调用 5 万、小模型占比 60%）

- 月节省：$12,150
- 回本时间：**~5.8 个月**

### 不值得做的阈值

- 月 Claude 账单 < $3000：训练投入 > 年节省，不划算
- 月 Claude 账单 $3000-$10000：需要严格成本控制才有意义
- 月 Claude 账单 > $10000：值得
- 月 Claude 账单 > $30000：**优先级最高**

## 隐性成本（容易忽视）

| 项 | 月成本（隐性） |
|---|---|
| 模型迭代维护（每季度 1 轮） | 0.5 人月 |
| 监控 / on-call | 0.2 人月 |
| 业务方抽检 / 反馈 | 0.2 人月 |
| Benchmark 集维护 | 0.1 人月 |
| GPU 电费 + 机房 | 本地 GPU 才算 |

折算约：1 人月/月 ≈ $10000（按高级工程师）。

## 长期优化杠杆

1. **提升小模型占比**：80% → 95%，月省 $6000
2. **推理优化**：vLLM + 量化（INT8/FP8）提升吞吐 2-3x，同等流量少卡
3. **批处理**：合并请求，提升单卡 QPS
4. **Spot 实例**：训练用 spot / preemptible，省 50-70%
5. **地区选择**：海外 GPU 云便宜，但要考虑合规

## 保守估算建议

对外报告时：
- 节省比例打 7 折（82% → 57%）
- 回本时间 × 1.5（2.3 月 → 3.5 月）
- 保留 Claude fallback 预算 120%

因为：
- 灰度期间会有 fallback 飙升期
- 迭代会临时降低小模型占比
- 新业务场景会先走 Claude

宁可保守估算过早达标，也别承诺过高。
